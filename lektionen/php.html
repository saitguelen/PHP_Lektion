🐘 PHP: Temellerden Ustalığa

Sunucu Taraflı Betik Dili

Web'in bel kemiğini oluşturan teknolojiyi adım adım öğrenelim.

🤔 Neden PHP?

🌐 Yaygın Kullanım

İnternet'teki sitelerin ~%78'i PHP kullanır. WordPress, Wikipedia, Facebook gibi devler PHP ile başlamıştır.

🚀 Kolay Öğrenim

Sözdizimi (syntax) C'ye benzer ve öğrenmesi kolaydır. Geniş bir topluluğa ve zengin bir dokümantasyona sahiptir.

🛠️ Geniş Ekosistem

Laravel, Symfony gibi modern framework'ler ve Composer gibi paket yöneticileri ile profesyonel uygulamalar geliştirmek çok kolaydır.

⚙️ Nasıl Çalışır?

PHP, sunucu taraflı bir dildir. Kodlar kullanıcının bilgisayarında değil, sunucuda çalışır ve tarayıcıya saf HTML olarak gönderilir.

👨‍🍳

Analoji: Restoran. Siz (kullanıcı) sipariş verirsiniz. Garson (tarayıcı) siparişi mutfağa (sunucu) iletir. Aşçı (PHP) yemeği (dinamik içerik) hazırlar ve garson size sadece sonucu (HTML sayfası) sunar.

💻 (İstek) → 🌐 → 🖥️[Apache + 🐘PHP] → 🌐 → 💻 (Cevap HTML)

📝 Temel Sözdizimi

PHP kodu <?php ... ?> etiketleri arasına yazılır. Her komut ; ile biter.

  echo "Merhaba Sait!"; // Ekrana yazı yazdırmak için kullanılır.
?>
YapıAçıklama

// ...Tek satırlık yorumlar için kullanılır.

📦 Değişkenler ve Veri Tipleri

Değişkenler, verileri saklamak için kullanılan kaplardır ve $ işareti ile başlarlar.

$isim = "Sait";                  // String (Metin)
$yas = 30;                       // Integer (Tam Sayı)
$ogrenciMi = true;               // Boolean (true/false)
$dersler = ["Java", "PHP"];      // Array (Dizi)
$bosDeger = null;                // NULL (Boş)

PHP, "gevşek tipli" (loosely typed) bir dildir. Değişkenin tipini önceden belirtmenize gerek yoktur.

🔍 Değişken Kontrolü: isset(), empty()

Değişkenlerin durumunu kontrol etmek için hayati fonksiyonlardır, özellikle formlardan gelen verilerle çalışırken.

isset($var)

Bir değişkenin tanımlı olup olmadığını ve değerinin null olmadığını kontrol eder. Tanımlıysa true döner.

$a = "test"; isset($a); // true
$b = null; isset($b); // false
isset($c); // false (tanımsız)

empty($var)

Bir değişkenin "boş" olup olmadığını kontrol eder. "", 0, "0", null, false, [] gibi değerler boş kabul edilir.

$x = ""; empty($x); // true
$y = 0; empty($y); // true
$z = "a"; empty($z); // false

🚦 Kontrol Yapıları: if/else

Programın akışını belirli koşullara göre yönlendirir.

$not = 75;
if ($not >= 85) {
  echo "Pekiyi";
} elseif ($not >= 70) {
  echo "İyi";
} else {
  echo "Geçer";
}

🔁 Döngüler: for & foreach

Bir kod bloğunu tekrarlamak için kullanılır.

for Döngüsü

for ($i = 1; $i <= 3; $i++) {
  echo "Sayı: $i <br>";
}

foreach Döngüsü

$dersler = ["Java", "PHP"];
foreach ($dersler as $ders) {
  echo $ders . "<br>";
}

📊 Diziler (Arrays)

Birden fazla değeri tek bir değişkende saklamamızı sağlar.

İndeksli Diziler

$arabalar = ["BMW", "Mercedes", "Audi"];
echo $arabalar[0]; // Çıktı: BMW

İlişkisel Diziler

$kisi = [
  "ad" => "Sait",
  "yas" => 30
];
echo $kisi["ad"]; // Çıktı: Sait

🔧 Sık Kullanılan Dizi Fonksiyonları

$dizi = ["elma", "armut"];

// Eleman sayısını verir
count($dizi); // 2

// Diziye eleman ekler
array_push($dizi, "çilek");

// Dizinin son elemanını çıkarır
array_pop($dizi);

// Diziyi sıralar
sort($dizi);

// Diziyi bir metne dönüştürür
$metin = implode(", ", $dizi); // "elma, armut"

// Metni diziye dönüştürür
$yeniDizi = explode(", ", $metin);

🧩 Fonksiyonlar

Tekrar tekrar kullanabileceğimiz kod bloklarıdır. "Bir kere yaz, her yerde kullan."

// Fonksiyon tanımı
function selamla($isim = "Misafir") {
  return "Merhaba, " . $isim . "!";
}

// Fonksiyonu çağırma
echo selamla("Sait"); // Merhaba, Sait!
echo selamla();      // Merhaba, Misafir!

📨 Form Verilerini İşleme: GET & POST

PHP, HTML formlarından gelen verileri $_GET ve $_POST süper global dizileriyle yakalar.

// form.html
<form action="islem.php" method="post">
  <input type="text" name="kullanici_adi">
  <button type="submit">Gönder</button>
</form>

// islem.php
if (!empty($_POST["kullanici_adi"])) {
    echo "Hoş geldin, " . htmlspecialchars($_POST["kullanici_adi"]);
}

Önemli: Güvenlik için kullanıcıdan gelen veriyi ekrana basarken daima htmlspecialchars() kullan!

🍪 Cookie ve Session Yönetimi

Kullanıcı bilgilerini ve durumunu (örneğin giriş yapıp yapmadığını) takip etmek için kullanılır.

Cookie (Çerez)

Kullanıcının tarayıcısında saklanır. "Beni Hatırla" gibi özellikler için idealdir.

// 1 saat geçerli bir cookie oluşturur
setcookie("kullanici", "Sait", time() + 3600);
// Cookie'yi okuma
echo $_COOKIE["kullanici"];

Session (Oturum)

Sunucuda saklanır. Güvenli veriler (kullanıcı ID'si gibi) için kullanılır. Daha güvenlidir.

session_start(); // Her zaman en başta olmalı!
// Session'a veri atama
$_SESSION["user_id"] = 123;
// Session'dan veri okuma
echo $_SESSION["user_id"];

📁 Dosya İşlemleri: Okuma ve Yazma

📅 Tarih ve Zaman Fonksiyonları

Tarih ve zamanı formatlamak ve işlemek için kullanılır.

// Şu anki tarihi ve saati gösterir
echo date('d-m-Y H:i:s'); // Örn: 29-08-2025 14:22:38

// Zaman damgası (timestamp) oluşturma
$yarin = mktime(0, 0, 0, date("m"), date("d")+1, date("Y"));
echo date('d-m-Y', $yarin);

// Daha modern yaklaşım: DateTime objesi
$date = new DateTime();
$date->modify('+1 day');
echo $date->format('d-m-Y');

🧮 Matematik Fonksiyonları

Sık kullanılan matematiksel işlemler için hazır fonksiyonlar.

// Rastgele sayı üretir
echo rand(1, 100);

// Sayıyı yuvarlar
echo round(4.7); // 5
echo round(4.3); // 4

// En büyük ve en küçük değeri bulur
echo max(5, 9, 2, 22); // 22
echo min(5, 9, 2, 22); // 2

// Mutlak değer
echo abs(-15); // 15

🚗 OOP Giriş

OOP, kodu nesneler (objects) ve sınıflar (classes) etrafında düzenleyen bir programlama yaklaşımıdır.

🏠

Analoji: Bir Sınıf (Class), bir evin mimari planıdır. Bir Nesne (Object) ise bu plana göre inşa edilmiş gerçek bir evdir.

class Araba {
  public $renk; // özellik (property)
  
  public function __construct($renk) {
    $this->renk = $renk;
  }

  public function calistir() { // metot (method)
    return "Araba çalıştı!";
  }
}

$bmw = new Araba("Siyah"); // Nesne oluşturma
echo $bmw->renk; // "Siyah"

🏛️ OOP Temel Kavramları

Kapsülleme (Encapsulation)

Verileri ve metotları bir sınıf içinde birleştirip dış erişimi kısıtlamak (public, private, protected).

Kalıtım (Inheritance)

Bir sınıfın başka bir sınıftan özellik ve metotları miras alması (extends).

Çok Biçimlilik (Polymorphism)

Farklı sınıfların aynı isimdeki metotları farklı şekillerde uygulaması.

➕ include ve require

Kodunuzu daha modüler hale getirmek için bir PHP dosyasına başka bir PHP dosyasını dahil etmenizi sağlar.

include

Dosya bulunamazsa bir uyarı (warning) verir ve kod çalışmaya devam eder.

// header.php
<h1>Siteme Hoş Geldiniz</h1>

// index.php
include 'header.php';
echo "Ana sayfa içeriği.";

require

Dosya bulunamazsa ölümcül bir hata (fatal error) verir ve kod durur. Genellikle kritik dosyalar (veritabanı bağlantısı gibi) için kullanılır.

// db_config.php - veritabanı ayarları
require 'db_config.php';
// ... veritabanı işlemleri

include_once ve require_once versiyonları, bir dosyanın sadece bir kez dahil edilmesini garantiler.

🗄️ Veritabanı Bağlantısı: PDO

Modern PHP'de veritabanı işlemleri için PDO (PHP Data Objects) kullanılır. Güvenli ve esnektir.

try {
  $conn = new PDO("mysql:host=localhost;dbname=test", "root", "");
  $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

  // SQL Injection'a karşı güvenli yöntem
  $stmt = $conn->prepare("SELECT * FROM users WHERE id = :id");
  $stmt->execute(['id' => 123]);
  $user = $stmt->fetch();

} catch(PDOException $e) {
  echo "Bağlantı hatası: " . $e->getMessage();
}

📦 Composer - Bağımlılık Yöneticisi

Java'daki Maven veya Node.js'teki NPM gibidir. Projenizin ihtiyaç duyduğu kütüphaneleri (paketleri) yönetir.

🛒

Analoji: Bir yemek tarifi (projeniz) için gerekli malzemelerin (kütüphaneler) listesini (composer.json) hazırlarsınız. composer install komutuyla tüm malzemeleri sizin için alır ve mutfağınıza (vendor klasörü) getirir.

✨ Modern PHP: Namespaces & Autoloading

Büyük projelerde kod organizasyonu için kritik öneme sahiptir.

İsim Alanları (Namespaces)

Aynı isme sahip sınıfların çakışmasını önler. Bir nevi sanal klasör yapısıdır.

namespace App\Controllers;
class UserController { }

Otomatik Yükleme (Autoloading)

Composer sayesinde sınıflar ihtiyaç duyulduğunda otomatik olarak yüklenir. require kalabalığı biter.

use App\Controllers\UserController;
$controller = new UserController();
